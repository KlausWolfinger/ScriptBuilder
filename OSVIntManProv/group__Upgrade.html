Detailed Description

Upgrade and Remote Patching Definitions and Methods.

Remote Patching capability allows semi-automating of the upgrade process and allows a remote client to initiate this action. The requests are forwarded to the Upgrade Library for processing.

int unsp__UpgradeFallback ( struct unsp__UpgradeFallbackRequest  FallbackRequest,
struct unsp__UpgradeFallbackResult *  FallbackResult 
)

This method performs a fallback of the most recent upgrade attempt.

It is used if errors are reported or manual verification checks fail on the upgraded system. Fallback will (attempt to) uninstall the upgrade on both nodes, if applicable.

Parameters
FallbackRequestAPI request
FallbackResultAPI result
unsp__UpgradeSecondNode ( struct unsp__UpgradeSecondNodeRequest  UpgradeSecondNodeRequest,
struct unsp__UpgradeSecondNodeResult *  UpgradeSecondNodeResult 
)

This method initiates an upgrade on the second node of a system.

Normally this operation would be executed after the first node has been upgraded successfully. No data parameters are included in the request, since all necessary information has already been sent via the unsp__Upgrade() method.
