enum unsp__KeysetUse

Describes whether a DN is used for Keyset Operation, and if so, in what way.

Enumerator
KeysetUnassigned 

Unassigned.

KeysetNone 

Indicates that the subscriber DN is not used for Keyset Operation and no additional keyset-related attributes will apply.

KeysetPrimaryLine 

Indicates that the subscriber DN is used as the Primary Line for a keyset, where primary line is defined as the first line to register from a given SIP endpoint and the line used by default when no secondary line key is pressed.

KeysetPhantomLine 

Indicates that the subscriber DN is used as a Phantom Line for one or more keyset where Phantom Line is defined as a line that does not appear as a Primary Line on any keyset, but may have an appearance as a Secondary Line on one or more keysets.

enum unsp__Subscribed

Indicates the action to be taken for the service feature.

Enumerator
Ignore 

Skip this feature/no action.

AddReplace 

Add this feature and/or replace any existing definition of the feature that the subscriber might have.

When a feature is AddReplace, the complete feature-specific data must be specified; missing fields will be assigned with their default values.

Inherited 

Indicates that the feature data are inherited by the Feature Profile assigned to the object.


Read Only

Delete 

Remove the feature from the subscriber.

Inherited features may not be removed, although they may be denied.

Denied 

Deny this feature to the subscriber.

Do not allow the subscriber to inherit the feature from a Feature Profile.

SwitchWide 

Indicates that the feature is set switch wide and thus all subscribers have access to it.


Read Only

Modify 

The feature must already exist from either being previously provisioned at the Feature Profile Level (and inherited) or provisioned directly.

Only the fields listed are modified. The local presence fields may be sent to reset overrides for fields.
